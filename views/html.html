HTML 5

El mito del 5

Hyper Text Markup Language, la base de absolutamente todo lo que encontramos en la web.

Qué es ?

Es el lenguaje de "Markup" utilizado para construir páginas web. Markup se refiere a una manera de organizar texto que sea distinguible del texto escrito usual, con el que se escribiría una obra literaria por ejemplo. En el caso de HTML, está constituido por Etiquetas HTML.

Consideren cada etiqueta como una pieza de lego que va construyendo toda una estructura. Usualmente, cada etiqueta va emparejada, como un <h1>, pero también existen etiquetas que no tienen pareja consideradas como etiquetas vacías. Un ejemplo de esto son las etiquetas <img>

Ejemplos de etiquetas y su uso habitual son:

						
<h1>Título</h1>
<h2>Subtítulos</h2>
.
.
.
<h6>Más subtítulos</h6>
						
					
						
<div>Un contenedor</div>
<p>Párrafos</p>
<img src="img/urlDeLaImagen.jpg" alt="Texto alternativo">
Para que el texto se vea como se vería en un editor de código, se usa PrismJS.

Hay muchas más etiquetas y de hecho, una de las principales diferencias entre HTML y HTML5 es el tipo de etiquetas que son válidas y el uso que se les da.

HTML vs HTML5

Lo primero que hay que entender es que HTML5 existe desde hace 4 años. Si, 4 años. Esto no es nada nuevo y por tanto, debería dejar de considerarse como algo innovador o importante. De hecho, nosotros nunca conocimos ninguna implementación de HTML por debajo de HTML4 y realmente, es irrelevante.

Doctypes

Doctype es lo primero que debe existir dentro de un documento html para asegurar que sea interpretado correctamente por parte de un navegador. Antes de HTML5, había distintos tipos de Doctypes. Esto se debía a que HTML era basado en SGML (Standard Generalized Markup Language) y por tanto necesitaba de un DTD (Document Type Definition). Así, en los antiguos tiempos oscuros existía lo siguiente:

				
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
o
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd">

Y dependiendo del tipo de Doctype que se declarara, los navegadores iban a interpretar el documento HTML de distintas maneras. Por fortuna para nosotros, hoy en día solo existe lo siguiente: 

				
<!doctype html>

Esto se debe a que HTML 5 no está basado en SGML y por lo tanto no necesita una definición que le permita a un navegador interpretarlo, solo tiene una interpretación.

Semántica

La semántica se refiere a que lo que está escrito refleja realmente su significado. En HTML, esto se representa por medio de los tags que significan algo. Entre esos, están los que les mostré anteriormente. También existen <div> y <span que son semánticamente neutrales.

HTML5 no introdujo las etiquetas semánticas (solo un par), pero si representa un momento en el tiempo en el que CSS evolucionó de tal manera que las etiquetas de presentación como <font> se volvieron innecesarias. Es por esto que muchas veces escucharan decir que HTML5 es "más semántico". Es mentira, pero es verdad a la vez.

Los elementos más relevantes que introdujo HTML5 son:

  • Elementos semánticos como: <header>, <footer>, <article> y <section>
  • Atributos de Control para formularios como: number, date, time, calendar y range
  • Elementos gráficos: <svg> y <canvas>
  • Elementos multimedia: <audio> y <video>

Además de esto, HTML5 introduce unos cuantos API's interesantes, que son:

  • HTML Geolocation
  • HTML Drag and Drop
  • HTML Local Storage
  • HTML Application Cache
  • HTML Web Workers
  • HTML SSE
Local Storage es maravilloso y un reemplazo perfecto para las cookies.
