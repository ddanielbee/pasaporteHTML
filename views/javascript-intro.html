Javascript - Introducción

La inyección de interacción en la web

Javascript es el lenguaje propio de los navegadores que nos permite interactuar con el DOM.

Qué es ?

Aparte del nombre y algunas similaridades en la sintáxis, Javascript no tiene nada que ver con Java. Vive en el cliente y sus implementaciones le permiten a scripts leídos por el cliente interactuar con el usuario, controlar el browser y comunicarse asincrónicamente con una base de datos. A pesar de que Javascript fuera pensado inicialmente solo para existir en el cliente (los navegadores), existe también la posibilidad de escribir código de servidor en Javascript, haciendo uso de lenguajes como Node.js.

El código en Javascript usualmente se ve así:

				
var foo = "bar";
foo += " foo"
console.log(foo);
El ejemplo de arriba escribiría "barfoo" en la consola.

Javascript puede ser escrito en el tag <head> o el <body> de un documento html y al igual que CSS hay varias formas de enlazarlo con un website.

Directamente en el código html:

				
<!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<title>Example</title>
	<script>
		var myTest = "Hi, I'm Javascript";
	</script>
</head>
<body>
	... content

	<script>
	var mySecondTest = "Hi, I'm also Javascript";
	</script>
</body>
</html>

O enlazando un archivo externo:

				
<!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<title>Example</title>
</head>
<body>
	... content
	<script src="js/myJsFile.js"></script>
</body>
</html>
Las buenas prácticas dictan que debemos enlazar los archivos de forma externa y no escribirlo directamente en el documento html. Además, es mejor si va al final del body. Excepciones a esta regla son: Modernizr, el api de Facebook y Google Analytics, entre otras.

Variables

Como cualquier otro lenguaje de programación, Javascript maneja variables. Estas pueden ser de los mismos tipos a los que ya estamos acostumbrados, pero tengan en cuenta que, a pesar de que existen tipos de variables, Javascript es lo suficientemente inteligente para saber qué tipo de variable es cada elemento sin que nosotros le digamos. Aquellos con familiaridad a Java, Objective-C y otros lenguajes, sabrán que esto no es siempre así. Los siguientes son ejemplos de variables.

				
var x = 5; 						// Un int.
var y = "5"; 					// Un string.
var z = 5.1; 					// Un float.
var foo = [x,y,z]; 				// Un array.
var bar = {x:5, y:"5", z:5.1};  // Un objeto.
var verdad = true; 				// un Boolean

// El siguiente bloque declara varias variables en un estilo de shorthand.
var a = "primera",
	b = "segunda",
	c = "tercera",
	d = "cuarta";

// Y una variable también puede ser declarada sin valor inicial.
var e; // undefined
e = "Ahora si tengo valor."
Especialmente en Javascript a nivel mundial se utiliza la convención "CamelCase" para escribir las variables, iniciándose la primera letra con minúscula. Así, una variable debería llamarse en el estilo "miVariableBonita" y no "Mivariablebonita". También en ocasiones se utilizan sub-guiones. El guión está reservado para la resta.

Operaciones

Igualmente, como en cualquier otro lenguaje, existen las operaciones. Los siguientes son los tipos de operadores que existen:

				
8 + 5; // Adición (13)
8 - 5; // Substracción (3)
8 * 5; // Multiplicación (40)
8 / 5; // División (1.6)
8 % 5; // Módulo (3)
8 ++;  // Incremento (9)
8 --;  // Decremento (7)
La suma también puede realizarse con strings

Algo importante a saber de Javascript, es que, ya que las variables no están definidas forzosamente, la suma entre un entero y un string es posible y tiene como resultado un string que contiene al valor entero.

Condicionales y Repetitivas

Los mismos tipos de condicionales y repetitivas de todos los demás lenguajes existen en Javascript. Primero el condicional, conocido como "if".

Un condicional cuenta de dos partes: la condición a analizar y la respuesta dependiendo del resultado de esa condición. Las condiciones siempre van a tener un resultado booleano, es decir, serán true o false, 1 o 0, no hay punto intermedio y dependen del tipo de operador de comparación que se utilice. Estos son los operadores de comparación y de paso los operadores lógicos que existen:

				
if(x == y) // Igualdad parcial (se compara el valor)
{
	// Hacer algo
}

if(x === y) // Igualdad total (se compara el valor y el tipo de variable)
{
	// Hacer otra cosa
}

if(x != y) // Desigualdad parcial (también hay total, igual que arriba)
{
	// bla
}

if(x < y) // Menor que (también hay mayor qué ">", menor igual que "<=" y mayor igual que ">=")
{
	// asdf
}

if(x == y && y == w) // Y, se deben cumplir ambas condiciones
{
	// qwer
}

if(x == y || y == w) // O, se debe cumplir al menos una de las condiciones
{
	//poiu
}
Nota: hay una forma avanzada de escribir condicionales que pueden encontrar en librerías o plugins que es la siguiente:
				
if(x == y)?"Acción cuando es cierto":"Acción cuando es falso";

// Esto es lo mismo que escribir

if(x == y)
{
	"Acción cuando es cierto";
} else
{
	"Acción cuando es falso";
}
La condición else está ahí para actuar con el resultado negativo de nuestra comparación inicial.

Finalmente, existe el comparador "else if" y el "switch", que van así:

if(x == y)
{
	// Si es cierto
} else if(x == z)
{
	// Si esto otro es cierto
} else
{
	// Si ninguna de las anteriores es cierta
}

// Switch
switch(x) {
    case 1:
        // Código que se ejecuta si x es 1.
        break;
    case 2:
        // Código que se ejecuta si x es 2.
        break;
    default:
        // Código que se ejecuta si x es otro valor cualquiera.
}

Repetitivas

Una repetitiva es un bloque de código que se debe repetir mientras se cumpla una condición. Existen 3 tipos de repetitivas que son for, while y do-while

				
// For
for(i == 0; i<10; i++)
{
	// Hacer algo
}

// Para un i que empieza en 0, mientras sea menor a 10 e incrementándose en 1 en cada vuelta que de la repetitiva, se hace lo que va dentro.

// While
while(i <10)
{
	// Hacer algo
	i++;
}

// Mientras i sea menor a 10, se hace lo que va dentro. El resultado sería el mismo. 

// Do-While
do
{
	// Hacer algo
	i++;
}
while(i < 10);

// Se hace algo la primera vez y se repite mientras i sea menor a 10. La diferencia con el while, es que aquí la acción se hace al menos una vez, así la condición sea falsa desde un comienzo.
Tengan cuidado con las repetitivas. La condición tiene que dejar de cumplirse en algún momento, o todo estallará.

Finalmente, un método realmente útil para el desarrollo web es "console.log()". Esto nos mostrará en la consola del navegador lo que vaya dentro de los paréntesis.
